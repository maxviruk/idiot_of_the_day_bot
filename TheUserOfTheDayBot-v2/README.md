# TheUserOfTheDayBot v2 (SQLite)

Телеграм-бот, который выбирает «красавчика дня» и «неудачника дня» в чате и ведёт статистику.
Это улучшенная копия [DevDmitryN/TheUserOfTheDayBot](https://github.com/DevDmitryN/TheUserOfTheDayBot)
с исправлением открытых issues, историей победителей и переходом на **SQLite** (отдельный сервер БД не нужен).

## Что нового по сравнению с оригиналом

### Прочие улучшения
- **SQLite вместо MySQL** — вся база в одном файле (`bot.db`), отдельный сервер не нужен, таблицы создаются сами.
- Новый Telegram Bot API (6.8.0), без устаревшего `ApiContextInitializer`.
- Токен вынесен в `config.properties` / переменные окружения (раньше был в коде).
- Устойчивость к не-текстовым сообщениям и неизвестным командам (раньше бот падал).
- Исправлен баг с определением «сегодня» (был формат `"DD"` — день года вместо дня месяца).
- Статистика сортируется по убыванию.

### История победителей
- Таблица `history` сохраняет каждого победителя с датой и типом игры.
- Команда `/history` показывает последние 20 записей.

## Команды
```
/reg         — вступить в игру
/unreg       — выйти из игры
/run         — разыграть «красавчика дня»
/loser       — разыграть «неудачника дня»
/stat_user   — статистика красавчиков
/stat_loser  — статистика неудачников
/history     — история победителей
/remove      — (админ, ответом на сообщение) удалить игрока
/reset       — (админ) сбросить статистику чата
/help        — справка
```

## Установка и запуск

1. **Конфиг.** Скопируйте пример и заполните токен/username:
   ```bash
   cp config.example.properties config.properties
   ```

2. **Сборка:**
   ```bash
   mvn clean package
   ```

3. **Запуск:**
   ```bash
   java -jar target/TheUserOfTheDayBot.jar
   ```

При первом запуске рядом появится файл `bot.db` — это и есть вся база.
Никаких MySQL, логинов и паролей. Бэкап = скопировать `bot.db`.
`config.properties` должен лежать рядом с jar (или укажите путь через `CONFIG_PATH`).

## Хостинг
Боту не нужны вебхуки и публичный домен (работает на long polling — сам стучится в Telegram).
Достаточно любого места, где постоянно запущен Java-процесс: дешёвый VPS, либо PaaS
с деплоем из GitHub. Поскольку база — это файл, на хосте нужно лишь обеспечить, чтобы
`bot.db` сохранялся между перезапусками (на VPS — по умолчанию; на PaaS — постоянный диск/volume).

## Перенос данных со старого MySQL-бота
Если у вас уже крутился оригинал на MySQL и нужно перенести статистику в SQLite —
это разовая выгрузка из MySQL и заливка в SQLite (движки разные, простым SQL-скриптом
не обойтись). Напишите — подготовлю скрипт конвертации под вашу старую базу.
